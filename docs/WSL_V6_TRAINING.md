# WSL Native Training For Gemma TIMMY V6

This project uses WSL as the preferred local training runtime. Do not reuse a Windows virtual environment in WSL; Python virtual environments contain OS-specific binary wheels.

## Standard Workspace

Use these two WSL paths:

```text
/workspace/martha
/workspace/TimmyBot
```

They should point to:

```text
/mnt/c/Users/Bot/Desktop/martha
/mnt/c/Users/Bot/Desktop/TimmyBot
```

Set them up from WSL:

```bash
cd /mnt/c/Users/Bot/Desktop/martha
sudo bash scripts/setup_wsl_workspace.sh
```

Verify:

```bash
ls -la /workspace
```

Expected:

```text
martha   -> /mnt/c/Users/Bot/Desktop/martha
TimmyBot -> /mnt/c/Users/Bot/Desktop/TimmyBot
```

## Create The WSL Virtual Environment

From WSL:

```bash
cd /workspace/martha

python3 -m venv .venv_wsl
source .venv_wsl/bin/activate

python -m pip install --upgrade pip setuptools wheel
python -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
python -m pip install -r requirements.txt
python -m pip install unsloth
```

Verify CUDA:

```bash
python - <<'PY'
import torch
print(torch.__version__)
print(torch.version.cuda)
print(torch.cuda.is_available())
print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else "no gpu")
PY
```

Run:

```bash
cd /workspace/martha
```

Example:

```bash

UNSLOTH_OUTPUT_DIR=outputs/v6/models/gemma_timmy_mldl_math_lora_exp2 \
bash scripts/train_v6_wsl.sh
```

## Training Lock

The script creates:

```text
outputs/training.lock
```

This prevents Timmy and Codex from starting two GPU training jobs at once. If training is interrupted, check for active jobs first:

```bash
pgrep -af 'train_gemma|unsloth|torchrun|accelerate|python'
nvidia-smi
```

If no training process is active, remove the stale lock:

```bash
rm -f /workspace/martha/outputs/training.lock
```

## TimmyBot Access

TimmyBot can trigger the same controlled script:

```bash
wsl bash -lc 'cd /workspace/martha && bash scripts/train_v6_wsl.sh'
```

Do not let Timmy execute arbitrary shell commands for training. It should call this narrow script or a wrapper that enforces the same lock file.
## V6.1 Consultant Training

V6.1 uses the consultant-style dataset generated by local Ollama Gemma and
supervised by schema validation:

```bash
cd /workspace/martha
bash scripts/train_v61_wsl.sh
```

Default V6.1 training settings:

- `UNSLOTH_TRAIN_DATA=outputs/v61/data/v61_consultant_combined_train_chat.jsonl`
- `UNSLOTH_OUTPUT_DIR=outputs/v61/models/gemma_timmy_martha_v61_consultant_lora`
- `UNSLOTH_MAX_SEQ_LENGTH=1024`
- `UNSLOTH_BATCH_SIZE=1`
- `UNSLOTH_GRAD_ACCUM=4`
- `UNSLOTH_NUM_EPOCHS=3`
- `UNSLOTH_LEARNING_RATE=1e-5`
- `UNSLOTH_LR_SCHEDULER=cosine`
- `UNSLOTH_MAX_GRAD_NORM=1.0`
- `UNSLOTH_SEED=42`

Use native WSL paths such as `/workspace/martha/...` while training. Avoid
training directly from `/mnt/c/...` to reduce cross-OS filesystem latency.

For a larger V6.1 production run, generate more accepted rows first. The current
combined V6.1 pilot is useful for smoke training, not final release training.
